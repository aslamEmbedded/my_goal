PHASE 02 C Mastery + Diploma YOU ARE HERE [ EXPAND ]
Pointers Structs Dynamic Memory Bitwise Ops File I/O CLI Projects
Deep C. This phase makes or breaks your embedded career. Every bug in firmware traces back to C you didn't understand.
// Topics — master all of these
  • Pointers — address, dereference, pointer arithmeticint *p; p++; *p = 5; — know exactly what happens
  • Pointers to pointers — **ptr, array of pointersUsed in argv, linked lists, 2D arrays
  • Structs — declaration, typedef, nested structsModel real hardware: typedef struct { uint8_t addr; uint8_t data; } I2C_Msg_t;
  • Struct padding & alignmentCritical: sizeof() surprises in embedded. Use __attribute__((packed))
  • malloc / calloc / realloc / freeWhen to use on MCU: sparingly — prefer static allocation
  • Stack vs Heap — where variables liveLocal vars = stack, malloc = heap, global = BSS/data segment
  • Bitwise — AND, OR, XOR, NOT, shiftsSET: reg |= (1<<n) CLR: reg &= ~(1<<n) TOGGLE: reg ^= (1<<n)
  • Bit fields in structsstruct { uint8_t flag:1; uint8_t mode:3; } — pack hardware register maps
  • Function pointersvoid (*callback)(void); — used in RTOS, HAL, event systems
  • String manipulation — no stdlib, manualImplement strlen, strcpy, strrev yourself
  • Arrays vs Pointers — not the same thingarr decays to pointer, but sizeof differs
  • const, volatile, static keywordsvolatile uint32_t *reg = 0x40020000; — hardware register access
